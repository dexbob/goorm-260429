페이지 열기 루트로 이동

RAG Pipeline Visualizer

텍스트를 넣어 청킹 → 임베딩 → 코사인 유사도 Top-K 검색까지 RAG 검색 준비 단계를 눈으로 따라가 보는 학습용 MVP입니다. 검색 후에는 LLM이 상위 검색 문맥을 합성해 답변 블록을 생성하고, 전체 청크 유사도 분포 차트도 함께 보여 줍니다.

실행 방법

  • .env.example을 참고해 OPENAI_API_KEY를 프로젝트 루트 .env에 넣습니다.
  • 개발: npm run dev — Vite(프론트)와 로컬 API(RAG_API_PORT, 기본 8789)가 함께 뜹니다.
  • 빌드: npm run build — 동일 디렉터리 index.html·assets/ 정적 허브용 출력이 갱신됩니다.
  • 프로덕션 단일 포트: npm run build && npm start — 미리 빌드된 정적 파일과 API를 한 프로세스에서 제공합니다.

필수 환경변수

  • OPENAI_API_KEY 또는 OPENROUTER_API_KEY: 임베딩 생성용
  • OPENROUTER_API_KEY: LLM 답변 합성용
  • OPENROUTER_MODEL (선택): 기본 mistralai/mistral-7b-instruct:free
검색 요청에는 클라이언트가 들고 있는 임베딩 벡터를 함께 보내 서버에서 질문 임베딩 후 유사도를 계산합니다. Vercel 서버리스처럼 인스턴스 간 메모리가 공유되지 않는 환경에서도 동작하도록 한 설계입니다.
